## Completed updates

### Platform foundation and organization

- Created separate public, student, and admin experiences, including login, dashboards, test management, test solving, results, student management, and leaderboards.
- Added the public landing page and expanded the student home, tests, scores, and leaderboard experiences with live data.
- Extracted large inline CSS and JavaScript blocks into dedicated `css/` and `js/` files, making page code easier to maintain without changing the product behavior.
- Kept the application framework-free and deployable as plain HTML, CSS, and JavaScript.

### Security and backend reliability

- Replaced legacy browser-side login behavior with Supabase Auth and profile-based role enforcement.
- Added protected-page role guards, Row Level Security policies, and authenticated database access patterns.
- Prevented students from receiving correct answers before submission by using a restricted student question view and server-side scoring/review functions.
- Added server-side student creation through the `create-student` Edge Function so service credentials do not enter frontend code.
- Added shared escaping, safe image URL, request, and formatting helpers for database-backed UI.
- Expanded the secure schema for profiles, tests, questions, assignments, attempts, answers, leaderboards, and question-image storage workflows.

### SAT test-taking and results

- Added test-day tools: answer elimination, hide/show timer, text sizing, keyboard shortcuts, a lazy-loaded Desmos calculator, and an SAT math reference sheet.
- Added student-produced response (SPR/grid-in) questions with server-side numeric-equivalence grading.
- Persisted in-progress answers, flags, eliminated choices, preferences, and per-question time data so students can resume an attempt.
- Added results analytics for Reading and Writing, Math, topic, difficulty, and time spent, including slow-question indicators.
- Added printable score reports and mobile improvements for the test-solving flow.

### Practice, progress, and motivation

- Added a secure “My Mistakes” practice flow with section/topic filters, server-graded checks, explanations after commitment, and cleared-mistake tracking.
- Added target-score setting, an attempt trend chart, study streaks, and rule-based achievement badges.
- Upgraded the leaderboard with test and time-period filters, deep-linked test selection, and a top-three podium.
- Added assignment due-date badges and urgency/overdue states on student pages.

### Full-length SAT content

- Added **SAT Practice Test A (Full-Length)** with 98 original questions:
  - 27 Reading and Writing questions in each of two modules.
  - 22 Math questions in each of two modules.
- Added automatic assignment of the test to all existing student profiles.
- Added an idempotent add-on that converts 10 Math questions—five per module—to SPR/grid-in format.

### Classes, Question Bank, and Vocabulary

- Added Classes for administrators to create groups, manage members, assign tests to a whole class, and set assignment due dates.
- Added a student learning-space view for class membership, assigned tests, due dates, and completion status.
- Added a secure Question Bank where students can build filtered practice sessions from questions they are authorized to access. Answers are graded on the server and session progress is recorded.
- Added an admin Question Bank browser covering published and draft test questions.
- Added admin-managed, class-scoped, and personal Vocabulary lists with word management, definitions, examples, synonyms, and antonyms.
- Added student flashcards, quizzes, personal word creation, and learning/known progress tracking.

### Design direction

- Restored the platform’s original teal/navy visual language after a redesign experiment.
- Restyled Classes, Question Bank, and Vocabulary to match the original page patterns.
- Preserved the alternative redesign in the `redesign-backup` branch for possible future reference.
